# Draft API docs

## Trigger

Creating or refreshing API documentation for current backend behavior.

## Workflow

1. Document endpoint summary and request parameters.
2. Add request and response examples.
3. Define error responses and status codes.
4. Specify authentication and authorization requirements.
5. Capture versioning and compatibility notes.

## Guardrails

- Keep examples consistent with actual implementation behavior.
- Prefer concise, copyable examples over long narrative text.

## Output

- Endpoint reference with examples
- Error and auth behavior notes
- Compatibility and versioning guidance
